# `<hive-progress>` — ミツバチが蜜を集めて巣に持ち帰る進捗表現

中央に吊られた**ハチの巣(ハニカム)**から、**ミツバチが採蜜に飛び立つ**。
花や画面の外で蜜を集め、**巣に戻ってきた瞬間に運んできた分だけ蜜(=表示%)が進む**。
巣のハニカムが下から黄金色に満ちていき、満タンで完了。

依存ゼロ(canvas のみ)。

## 進捗モデル(このコンポーネントの核)

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | 外部から流し込む唯一の入力 |
| `displayValue` | 巣に貯まった蜜 0–100 | ミツバチが持ち帰ると進む(読み取り専用) |

正直設計のキモは **「飛行中の全ミツバチが運ぶ蜜 + 貯蔵済み(committed)は決して `value` を超えない」** こと:

- 実進捗 `value` が「まだ巣に届いていない分」を持つと、ミツバチが
  `payload = min(CHUNK, value - committed)` を抱えて飛び立つ。
- ミツバチが**巣に入った瞬間**に `displayValue += payload`(`deposit` イベント)。
  つまり表示は実進捗に**遅れて**追従するが、**決して超えない・巻き戻らない**。
- `value` が 100 に達すると、残りを運び切るまでミツバチが往復し、
  巣が満ちて `complete` を1度だけ発火する(放置しても必ず完走)。

中央上の % は `displayValue`(集めた蜜)。**ミツバチが帰巣するたびにカクッと進む**のが
このバリアントの見どころで、これは「表示が実進捗に遅れる」正直な量子化として設計されている。

### ステップ増加は往復を待たず即反映

`value` が1フレームで `STEP_THRESHOLD`(=4)以上**一気に**増えたとき(10%刻みのステップ更新や、
サーバーから離散的に届く更新)は、ミツバチの往復を待たずに蜜面をその値まで**即座に**引き上げる
(`#stepFill`)。飛行中のミツバチが運んでいた分は二重計上を避けるため payload を 0 にして即反映へ
統合し、にぎやかしに数匹を「蜜を運ばない演出バチ」として送り出す。連続的(smooth)に少しずつ
増えるときは従来どおりミツバチが往復で運ぶ。どちらも `displayValue` は `value` を超えない。

## 採蜜のふるまい

- 巣から最大 `MAX_BEES` 匹が同時に飛ぶ。`SPAWN_COOLDOWN` ごとに1匹ずつ送り出す。
- 採蜜先は **画面内の花を優先**(1匹が1輪を予約=`claimed`、取り合わない)。到着して
  `GATHER_MS` ホバリングし、花粉を散らしてから巣へ帰る(帰り道は蜜玉を抱える)。
- **採蜜された花は萎んで消える**(`wilting`)。**画面内に花が無くなると、ミツバチは
  画面の外**(左右や上の縁を越えた先)へ蜜を集めに行く。
- ステアリング(到着減速＋ふらつき)で飛ぶので、毎回ゆらゆらと違う軌道になる。
- 巣のまわりには `AMBIENT` 匹の**見張りバチ**が常に舞っている(進捗とは無関係の演出)。

## 遊び(スコア・進捗には影響しない)

- 初期状態の花は **3輪**。**草むらをクリック/タップすると花が咲く**。
- **画面内の花から採蜜できたときだけ +100点**(`+100` のエフェクトとともにスコア加算)。
  画面外で集めた蜜は得点にならない。花を植えるほど稼げるが、`value` / `displayValue`
  には一切影響しない純粋な演出レイヤー。
- `.score` で現在の得点を読める。`reset()` で 0 に戻る。

## 使い方

```html
<script type="module" src="hive-progress.js"></script>

<hive-progress id="hv"></hive-progress>

<script>
  const hv = document.getElementById("hv");
  onServerProgress((pct) => { hv.value = pct; });

  hv.addEventListener("complete", () => { /* 満蜜 */ });
  hv.addEventListener("deposit", (e) => console.log("持ち帰り", e.detail));
</script>
```

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 460 | 高さpx(CSSの height でも可) |
| `demo` | なし | `smooth` / `step`(10%刻み)で自走。**本番では使わない** |
| `duration` | 16000 | demoの所要ms |

### プロパティ / メソッド / イベント

- `.value` / `.displayValue`(=巣に貯まった蜜%) / `.trips`(採蜜往復した回数) / `.score`(得点)
- `.steps` / `.note` — テキスト表示(全バリエーション共通)
- `.reset()` — 進捗・ミツバチ・蜜・花・得点を初期化
- イベント `complete` — 巣が満ちた(表示が100%)とき(1回)
- イベント `deposit` — ミツバチが蜜を持ち帰ったとき(`detail: { amount, collected }`)

## 実装ノート

- **committed 管理**: `#committed()` = 貯蔵済み + 飛行中ミツバチの payload 合計。
  送り出しは `value - committed` が残っている時だけ。これで表示が実進捗を超えない。
- **ハニカム**: セルは正規化座標(単位円内の六角格子)で一度だけ生成し、描画時に
  巣の半径 `R` でスケール。蜜面 `lineY` より下のセルを黄金色で塗る=下から満ちる。
- **蜜面の追従**: `displayValue`(持ち帰りで段階的に増える)へ `#honeyShown` をイージング
  追従させ、帰巣のたびに滑らかに一段上がるように見せる。
- **ミツバチ**: 速度ベクトル向きに胴体を回し、羽ばたきは時間で、ふらつきは進行方向の
  直交成分で表現。帰り道だけ蜜玉を描く。
- **依存ゼロ**: 物理エンジンは使わず、飛行はステアリング、蜜・花・花粉はイージングで表現。
