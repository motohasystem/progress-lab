# `<water-progress>` — 傾けて遊べる水位型プログレス

粒子物理(Matter.js)で水滴が注がれ、水位=進捗として満ちていくプログレス表現。
端末の傾きで水面と水滴の挙動が変わり、水槽内に手描きの障害物を置いて遊べる。

![concept] 実進捗(value)→粒の放出→着水→水位、という因果で表示が決まる。

## 進捗モデル(このコンポーネントの核)

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | 外部から流し込む唯一の入力。粒の**放出量**を決める |
| `landedValue` | 着水済み | 表示水位。物理遅延で value に追いつく(読み取り専用) |

- 表示は実進捗に**遅れる**ことはあっても**嘘をつかない**。放出済みの粒は必ず水になる。
- 完了フラッシュ: `value=100` から `flush-delay` ms 経過後、障害物に留まった粒も強制着水し、水位を必ず 100% に確定する。
- 数字表示(中央の%)は `value` をそのまま出す。演出と正確さを分離している。

## 使い方

```html
<script type="module" src="water-progress.js"></script>

<water-progress id="wp" toolbar></water-progress>

<script>
  const wp = document.getElementById("wp");

  // 実進捗を流し込むだけ(ポーリング・SSE・WebSocket 何でも)
  onServerProgress((pct) => { wp.value = pct; });

  wp.addEventListener("complete", () => { /* 表示も100%に到達 */ });
</script>
```

matter-js が未ロードなら cdnjs から自動取得する。バンドルする場合は先に
`window.Matter` を用意しておけばネットワークアクセスは発生しない。

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 400 | 高さpx(CSSの height でも可) |
| `toolbar` | なし | ペン/消しゴム/傾け/全消去/センサーUIを表示 |
| `interactive` | toolbar有→`draw` / 無→`tilt` | ポインタ操作。`none` で無効 |
| `demo` | なし | `smooth` / `step`(10%刻み)で自走。**本番では使わない** |
| `duration` | 16000 | demoの所要ms |
| `drop-gain` | 0.09 | 1粒あたりの進捗%。小さいほど粒が増える |
| `flush-delay` | 3000 | 完了フラッシュまでの猶予ms |

### プロパティ / メソッド / イベント

- `.value` / `.landedValue` / `.tilt`(-35〜+35°)
- `.steps` / `.note` — テキスト表示(下記)
- `.reset()` — 進捗・水・粒を初期化(障害物は残す)
- `.clearObstacles()`
- `.enableTiltSensor()` — **必ずユーザー操作内で呼ぶ**(iOSの許可ダイアログ要件)。
  返り値 `"granted" | "denied" | "unsupported"`
- イベント `complete` — 水位が100%に到達(1回)

## テキスト表示("xxxをしています…")

画面**左上**に2種類のテキストを出せる。どちらも全バリエーション共通の仕組み。

**`.steps` — 進捗しきい値ごとの自動ラベル**(上段・アクセント色)

`value` から逆引きされる。進捗に正直なステップ表示。

```js
wp.steps = [
  { label: "接続を確立",   until: 8 },
  { label: "データ取得",   until: 30 },
  { label: "解析",         until: 55 },
  { label: "完了",         until: 100 },
];  // value <= until の最初の label が表示される
```

**`.note` — 進捗とは独立した任意の一行コメント**(下段・控えめ色)

しきい値に縛られず、設定した瞬間に表示される。空文字で消える。

```js
wp.note = "サーバーAと通信しています…";
// …
wp.note = "";  // 非表示
```

`note` は進捗に連動しないため、`value` が嘘をつかない設計とは別レイヤーの
自由な情報欄として使える(再試行中・対象ファイル名など)。

## 実装ノート(試行錯誤の記録 — 変更前に読むこと)

これらの値・構造は iPad Pro (Safari) 実機での試行錯誤の結果。

1. **iOSのresize問題**: Safariはツールバー伸縮のたびに `window.resize` を発火する。
   素朴に resize で初期化すると描いた障害物が消える。本実装は ResizeObserver
   (実サイズ変化時のみ発火)で構造的に回避している。
2. **センサー許可はiframe内で自動拒否**: サンドボックスiframe(プレビュー等)では
   `DeviceOrientationEvent.requestPermission()` がダイアログなしで denied になる。
   トップレベルのHTTPSページで使うこと。ローカル検証は `tailscale serve` などで
   HTTPS終端すると楽。
3. **物理はMatter.jsに任せる**: 自前のビットマップ衝突は、斜面での引っかかり・
   高速時のトンネリングが頻発した(サブステップ化でも質感が出ない)。
   剛体エンジン移行で全て解決した。
4. **摩擦値**: `friction: 0.001 / frictionStatic: 0.002` はほぼ下限。
   0 にすると水平面で永久微振動することがある。
5. **gooフィルタの閾値生存**: `blur(3.5)` × 描画半径 `×2.1` × アルファ行列 `18 -7`
   はセットで調整済み。blurを上げる/半径を下げると**単独で落下中の粒が閾値を
   下回って消える**。融合を強めたい場合は3つを同時に動かすこと。
6. **放出レート制限**: 1フレーム最大3粒。10%刻みの進捗報告が来ても約0.6秒の
   バーストに引き伸ばされ、カクカクした実進捗を滑らかな水位上昇に変換する。
7. **粒数上限700**: M4 iPadで余裕。低スペック端末向けには `drop-gain` を
   上げて粒数を減らすのが第一手。さらに必要なら水滴同士の衝突を
   collisionFilter で切る。

## 調整パラメータ一覧(ソース冒頭の定数)

| 定数 | 値 | 効果 |
|---|---|---|
| `GOO_BLUR` | 3.5 | 融合距離。上げると遠くの粒とくっつくが単独粒が消えやすい |
| `GOO_ALPHA` | "18 -7" | 閾値 ≈ 0.39。第2項を浅くすると生存しやすいが輪郭が緩む |
| `DROP_VIS_SCALE` | 2.1 | 描画半径倍率 |
| `WAVE_TENSION/SPRING/DAMP` | 0.18 / 0.018 / 0.985 | 水面の波の硬さ・復元・減衰 |
| `DROP_OPTS` / `OBSTACLE_OPTS` | — | Matter.jsの材質(摩擦・反発) |
