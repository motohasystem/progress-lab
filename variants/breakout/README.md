# `<breakout-progress>` — ブロック崩し型プログレスバーゲーム

実進捗が進むほどボールが画面中央から上へ次々と投入される。プレイヤーはパドルを
マウス/タッチで動かしてボールを跳ね返し、ブロックを崩していく。
崩したブロック数 = 表示進捗。**依存ライブラリなし(物理は自前)**。

## 進捗モデル

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | ボールの**投入数**を決める。さらに破壊の確定を頭打ちにする |
| `clearedValue` | 破壊済み | 表示進捗 = 破壊ブロック / 全ブロック(読み取り専用) |

正直契約の要点:

- ブロックは**最初からすべて破壊可能**。硬さ(HP)があり、ボールが当たるたび
  ひびが入り、HP=0 で割れる。
- ただし破壊の**確定**は実進捗で頭打ちにする。HP=0 になっても実進捗が追いつく
  までは砕けず、ひび割れたまま留まる。ゆえに `破壊済み ≤ value` となり、
  表示は実進捗を超えない。難易度は **ボール数 × 硬さ** で調整する。
- 放置対策が二重: ① 場外に落ちたボールは中央から**再投入**される(進捗を失わない)
  ② 完了フラッシュで残ブロックを**一斉破壊**(フィナーレ)。
- パドルは「進捗を早める/気持ちよくする」操作であり、進捗を生むわけではない。
  触らなくてもプログレスバーとして必ず完走する。

## ゲームの構造

```
中ほどからボールが上へ射出される → ブロックを削る(ひび → 破壊)
→ 落ちてきたボールをパドルで跳ね返して再び上へ → さらに削る
進捗が上がるほどボールが増え、砕ける数(破壊枠)も広がる
```

中央付近には実進捗の%が薄く表示される。右上は同時に飛んでいるボール数。

## 使い方

```html
<script type="module" src="breakout-progress.js"></script>
<breakout-progress id="bp"></breakout-progress>
<script>
  onServerProgress((pct) => { bp.value = pct; });
  bp.addEventListener("complete", () => { /* 表示も100%に到達 */ });
</script>
```

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 400 | 高さpx(CSSの height でも可) |
| `max-balls` | 28 | 同時に飛ぶボールの最大数(実進捗100%時) |
| `rows` | 6 | ブロックの段数。列数は幅から自動算出 |
| `brick-hp` | 2 | ブロックの硬さ=破壊に要する命中数(1–9) |
| `flush-delay` | 2500 | 完了フラッシュまでの猶予ms |
| `demo` | なし | `smooth` / `step`(10%刻み)で自走。**本番では使わない** |
| `duration` | 18000 | demoの所要ms |

### プロパティ / メソッド / イベント

- `.value` / `.clearedValue`
- `.steps` / `.note` — テキスト表示("xxxをしています…"。全バリエーション共通)
- `.reset()` — 進捗・ボール・ブロックを初期化し、壁を組み直す
- イベント `complete` — 表示進捗が100%に到達(1回)

## 実装ノート

- **依存ゼロ**: 円とAABBの反射なので物理エンジンは使わず自前実装。water/tnt と違い
  ネットワークアクセスが一切発生しない。
- **トンネリング防止**: 高速時にブロックをすり抜けないよう、ボールの移動を
  `BALL_R * 1.6` 以下のサブステップに分割して各ステップで衝突判定する。
- **HP削りと破壊確定の分離**: ボールはどのブロックにも当たれば必ず反射し、HPを
  1削る(ひび)。HP=0でも破壊が確定するのは `破壊済み < 実進捗の枠` のときだけ。
  これにより「全ブロック破壊可能」という手触りを保ちつつ、表示が実進捗を超えない
  ことを保証する。HP=0で実進捗待ちのブロックは、進捗が進んだ瞬間に砕ける。
- **巻き戻し**: `value` が下がると、破壊枠を超えた分を破壊順の新しい方から復活
  (HPも戻す)、ボールも実進捗相応の数まで間引く(`core/progress-base.js` の
  `prev` で検知)。
- リサイズ時はブロックの破壊状態を保ったまま座標だけ再計算する(列数は初回固定)。

## 調整パラメータ(ソース冒頭の定数)

| 定数 | 値 | 効果 |
|---|---|---|
| `BALL_SPEED` | 4.3 | ボール速度(px/frame)。常に一定に保たれる |
| `BALL_R` | 5.5 | ボール半径px |
| `IDEAL_CELL` | 34 | 1ブロックの目標幅px。列数の算出に使う |
| `SPAWN_INTERVAL` | 150 | ボール投入の最小間隔ms。短いほど一気に増える |
| `RESPAWN_DELAY` | 650 | 場外ボールの再投入までのms |
| `FLUSH_PER_FRAME` | 3 | 完了フラッシュ時に1フレームで砕く残ブロック数 |
| `ROW_COLORS` | — | 段ごとのブロック色 |
