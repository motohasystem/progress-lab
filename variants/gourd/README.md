# `<gourd-progress>` — ひょうたんのつるが伸びる進捗表現

実進捗に合わせて、鉢から **ひょうたんのつる** が上へ伸びていく。つるの要所には
**ひょうたんの実** が生り、クリックすると **腰のくびれでぱかっと割れて** 中から
めでたいもの(小判・達磨・富士・宝珠・鏡餅・打ち出の小槌)が湧き出る。

依存ゼロ(canvas のみ)。

## 進捗モデル(このコンポーネントの核)

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | 外部から流し込む唯一の入力 |
| `displayValue` | つるの伸び | `value` へ滑らかに追従(読み取り専用) |

- 表示(つるの伸び)は実進捗へ**遅れて**追従するが**嘘はつかない**。`value` が
  巻き戻ればつるも縮む。
- 実の出現は「つるがそのしきい値まで伸びたら」起こる。**開封(クリック)は進捗に
  一切影響しない** — 表示は常に実進捗に正直。
- 完了フィナーレ: つるが100%に達して `finale-delay` ms 経過後、未開封の実を
  下から順に自動で割る。放置しても必ずめでたく締まる。

## 使い方

```html
<script type="module" src="gourd-progress.js"></script>

<gourd-progress id="gp"></gourd-progress>

<script>
  const gp = document.getElementById("gp");

  // 実進捗を流し込むだけ(ポーリング・SSE・WebSocket 何でも)
  onServerProgress((pct) => { gp.value = pct; });

  gp.addEventListener("complete", () => { /* つるも100%に到達 */ });
  gp.addEventListener("gourd-open", (e) => console.log(e.detail.treasure));
</script>
```

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 460 | 高さpx(CSSの height でも可) |
| `demo` | なし | `smooth` / `step`(10%刻み)で自走。**本番では使わない** |
| `duration` | 16000 | demoの所要ms |
| `finale-delay` | 3500 | 完了後、未開封の実を自動開封するまでの猶予ms |

### プロパティ / メソッド / イベント

- `.value` / `.displayValue`(=つるの伸び)
- `.steps` / `.note` — テキスト表示(下記、全バリエーション共通)
- `.openAll()` — 出現済みで未開封の実をすべて割る
- `.reset()` — 進捗・つる・実を初期化
- イベント `complete` — つるが100%に到達(1回)
- イベント `gourd-open` — 実が割れたとき。`detail: { treasure }`

## テキスト表示("xxxをしています…")

画面**左上**に2種類のテキストを出せる(全バリエーション共通の仕組み)。

**`.steps` — 進捗しきい値ごとの自動ラベル**(上段・アクセント色)

```js
gp.steps = [
  { label: "種まき",   until: 8 },
  { label: "つる伸び", until: 50 },
  { label: "結実",     until: 95 },
  { label: "豊作",     until: 100 },
];  // value <= until の最初の label が表示される
```

**`.note` — 進捗とは独立した任意の一行コメント**(下段・控えめ色)

```js
gp.note = "サーバーAと通信しています…";
gp.note = "";  // 非表示
```

## めでたいもの

実ごとに以下が順番に割り当てられる(`小判 → 達磨 → 富士 → 宝珠 → 鏡餅 → 小槌`)。
すべて canvas で手描き(**文字なしの絵柄のみ**)、後光と火花つき。割れると
大きくズームしてせり上がる。

| type | 絵柄 |
|---|---|
| `koban` | 小判(金の打目) |
| `daruma` | 達磨(両目入り=満願) |
| `fuji` | 富士山と日の出 |
| `houju` | 宝珠(火炎宝珠) |
| `kagami` | 鏡餅 |
| `mallet` | 打ち出の小槌 |

## 実装ノート

- **正直設計**: つるの伸び `displayValue` は `value` へイージング追従するだけで、
  先回りも水増しもしない。中央の % は `value` をそのまま表示する。
- **うねりはランダム**: つるの蛇行はランダム生成した倍音の合成(`#genWave`)。
  低い倍音ほど大きく振れて画面幅いっぱいにうねり、高い倍音で細かくグネグネする。
  合計振幅を1に正規化して全幅に収め、`reset()` ごとに引き直す(毎回違う形)。
- **実の幾何**: ひょうたんは「上の小さなふくらみ＋下の大きなふくらみ」。腰のくびれが
  自然な破断点なので、割れ演出はそこで上下に分離させている(`#gourdBulb`)。
- **当たり判定**: 実の中心と半径は描画のたびに更新し、pointerdown で距離判定。
  Shadow DOM 内 canvas の座標は CSS px(`ctx.scale(dpr,dpr)` 済み)。
- **依存ゼロ**: 物理エンジンは使わず、伸び・出現・割れはすべてイージングで表現。
