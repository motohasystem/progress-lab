# `<gourd-progress>` — ひょうたんのつるが伸びる進捗表現(お宝キャッチゲーム付き)

実進捗に合わせて、鉢から **ひょうたんのつる** が画面幅いっぱいにグネグネうねりながら
上へ伸びていく。つるの要所に **ひょうたんの実** が生り、**成るとすぐ腰のくびれで
ぱかっと割れて**、中のめでたいもの(小判・達磨・富士・宝珠・鏡餅・打ち出の小槌)が
**いったん上へ飛び上がってから(左右ランダムに散って)落下**する。各お宝は
**50〜500 点のランダム得点**で、**300 点以上はキラキラ・500 点はギラギラ**に輝く。
**カーソルの X方向に追従するキャラ(かごを掲げた子)**で受け止めると加点。
**最後に結果スコアを表示**し、落としたお宝を全部キャッチすると **PERFECT!** 演出。

依存ゼロ(canvas のみ)。

## 進捗モデル(このコンポーネントの核)

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | 外部から流し込む唯一の入力 |
| `displayValue` | つるの伸び | `value` へ滑らかに追従(読み取り専用) |

- 表示(つるの伸び)は実進捗へ**遅れて**追従するが**嘘はつかない**。`value` が
  巻き戻ればつるも縮む。
- 実の出現・開封・お宝の落下・キャッチ(得点)は**進捗に一切影響しない** —
  表示は常に実進捗に正直。ゲームは純粋に演出レイヤー。

## キャッチゲーム

- つるが各しきい値まで伸びると実が生り、`OPEN_DELAY`(約0.3秒)後に自動で割れる。
- お宝は割れた瞬間に**上向き初速**で飛び出し(左右の初速はランダム)、重力で落下。
  左右の壁では軽く跳ね返る。
- 各お宝は `POINT_TABLE` からランダムに **50〜500 点**。`300` 以上で**キラキラ**
  (回る星)、`500` で**ギラギラ**(脈動グロー＋回転する十字フレア)。
- キャラ(かご)は**カーソルの X方向にだけ**イージング追従(高さは下部に固定)。
  かごの口に落下中のお宝が重なるとキャッチし、得点ぶんの派手な `+N`(高得点ほど
  大きく・きらびやか)・衝撃波リング・星と紙吹雪・かごのバウンドが出る。
- 全部落とし切ると**結果スコア**をカウントアップ表示。取りこぼし `0` で全部
  キャッチすると **PERFECT!**(`perfect` イベント発火)。

## 使い方

```html
<script type="module" src="gourd-progress.js"></script>

<gourd-progress id="gp"></gourd-progress>

<script>
  const gp = document.getElementById("gp");

  // 実進捗を流し込むだけ(ポーリング・SSE・WebSocket 何でも)
  onServerProgress((pct) => { gp.value = pct; });

  gp.addEventListener("complete", () => { /* つるも100%に到達 */ });
  gp.addEventListener("gourd-open", (e) => console.log(e.detail.treasure));
  gp.addEventListener("perfect", () => console.log("全部キャッチ!", gp.score));
</script>
```

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 460 | 高さpx(CSSの height でも可) |
| `demo` | なし | `smooth` / `step`(10%刻み)で自走。**本番では使わない** |
| `duration` | 16000 | demoの所要ms |

### プロパティ / メソッド / イベント

- `.value` / `.displayValue`(=つるの伸び) / `.score`(得点・読み取り専用)
- `.steps` / `.note` — テキスト表示(下記、全バリエーション共通)
- `.openAll()` — 出現済みの実を即座に割って落とす
- `.reset()` — 進捗・つる・実・スコアを初期化
- イベント `complete` — つるが100%に到達(1回)
- イベント `gourd-open` — 実が割れたとき。`detail: { treasure }`
- イベント `perfect` — 落としたお宝を全てキャッチしたとき

## テキスト表示("xxxをしています…")

画面**左上**に2種類のテキストを出せる(全バリエーション共通の仕組み)。

**`.steps` — 進捗しきい値ごとの自動ラベル**(上段・アクセント色)

```js
gp.steps = [
  { label: "種まき",   until: 8 },
  { label: "つる伸び", until: 50 },
  { label: "結実",     until: 95 },
  { label: "豊作",     until: 100 },
];  // value <= until の最初の label が表示される
```

**`.note` — 進捗とは独立した任意の一行コメント**(下段・控えめ色)

```js
gp.note = "サーバーAと通信しています…";
gp.note = "";  // 非表示
```

## 実装ノート

- **正直設計**: つるの伸び `displayValue` は `value` へイージング追従するだけで、
  先回りも水増しもしない。中央上の % は `value` をそのまま表示する。
- **うねりはランダム**: つるの蛇行はランダム生成した倍音の合成(`#genWave`)。
  低い倍音ほど大きく振れて画面幅いっぱいにうねり、高い倍音で細かくグネグネする。
  合計振幅を1に正規化して全幅に収め、`reset()` ごとに引き直す(毎回違う形)。
- **実の幾何**: ひょうたんは「上の小さなふくらみ＋下の大きなふくらみ」。腰のくびれが
  自然な破断点なので、割れ演出はそこで上下に分離させている(`#gourdBulb`)。
- **落下の物理**: お宝は上向き初速で飛び出し、左右初速はランダム、重力で落下。
  壁で軽く反射。キャッチ判定はかご中心からの距離(`CATCH_R`)で、落下中(`vy>0`)のみ。
  調整値はソース冒頭の `OPEN_DELAY / GRAVITY / POP_UP / DRIFT / CATCH_R / POINTS`。
- **依存ゼロ**: 物理エンジンは使わず、伸び・出現・割れ・落下はすべて自前で表現。
