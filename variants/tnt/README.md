# `<tnt-progress>` — TNT爆破型プログレスバーゲーム

実進捗の分だけTNTブロックが投下され、土の地形に降り積もる。
タップで着火→爆発→誘爆チェーン。爆発は地面も掘る。
爆破済みブロック数=表示進捗。

## 進捗モデル

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | TNTブロックの**投下量**を決める(1個 = `block-gain` %) |
| `detonatedValue` | 爆破済み | 表示進捗。読み取り専用 |

正直契約の要点:
- **TNTの消滅は必ず進捗になる**。爆発・場外への吹き飛び、いずれも回収して加算。
- **土は風景**。爆発で破壊されるがクレーター演出であり、進捗には計上しない。
- 放置対策が二重: ①各ブロックは3.5〜7秒で**自然発火** ②**100%到達の瞬間、
  残ブロックをすべて即着火して一斉爆発**(フィナーレ)。触らなくても必ず完走する。
- **地面(土)ブロックを全部破壊し尽くすと花火が打ち上がり "Congratulations!" を表示**する
  (進捗100%やTNTの残数とは無関係。掘り切っている間ずっと上がる)。
- タップは「進捗を早める」操作であり、進捗を生むわけではない。
  投下されていないブロックは爆破できない=実進捗を超えられない。

## ゲームの構造

水槽の上部約9割(`dirt-fill`)まで土ブロックの地形が敷かれている(最上段は草)。

```
TNTが地表に積もる → 爆破でクレーターができる
→ 次に降るTNTがクレーターへ転がり込む
→ 深い穴でまとめて誘爆 → さらに掘れる
```

爆破のたびに地形が変わる採掘ゲーム。中央の%数字は地形より手前に表示され、
土に埋もれていても常に進捗を読み取れる。

## 使い方

```html
<script type="module" src="tnt-progress.js"></script>
<tnt-progress id="tp"></tnt-progress>
<script>
  onServerProgress((pct) => { tp.value = pct; });
  tp.addEventListener("complete", () => { ... });
</script>
```

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 400 | 高さpx |
| `block-gain` | 0.625 | 1ブロックあたりの% (0.625 → 全160個) |
| `dirt-fill` | 0.9 | 土を積む高さ(水槽の高さ比、0〜0.95) |
| `demo` | なし | `smooth` / `step` 自走(本番では使わない) |
| `duration` | 20000 | demoの所要ms |

### プロパティ / メソッド / イベント

- `.value` / `.detonatedValue`
- `.steps` / `.note` — テキスト表示(下記)
- `.reset()` — 進捗・TNT・演出を初期化し、**地形も敷き直す**
- `complete` — 表示進捗100%到達(1回)

### テキスト表示("xxxをしています…")

画面**左上**に2種類のテキストを出せる(全バリエーション共通の仕組み)。

- `.steps` — 進捗しきい値ごとの自動ラベル(上段)。`value` から逆引きされる。
  ```js
  tp.steps = [
    { label: "鉱脈を探索",   until: 40 },
    { label: "採掘中",       until: 90 },
    { label: "完了",         until: 100 },
  ];
  ```
- `.note` — 進捗に連動しない任意の一行コメント(下段)。設定した瞬間に表示、
  空文字で消える。
  ```js
  tp.note = "サーバーAと通信しています…";
  tp.note = "";  // 非表示
  ```

## ゲームデザインのメモ

- **誘爆チェーン**(半径60px・導火線120〜420ms)が爽快感の核。積もった山や
  穴に溜まった塊の1個をタップ→連鎖爆発、が気持ちいい。
- **爆風**(半径85px)は周囲のTNTを物理的に吹き飛ばし、**土の破壊**(半径64px)
  が地形を変える。タップ位置の選択が掘削の戦略になる。
- 投下ペースは `SPAWN_INTERVAL`(60ms)。10%刻みの進捗報告では16個が
  約1秒でドバッと降る。なめらか進捗では投下予算(spawned < value)が律速する
  ため、間隔を短くしても自然なペースに収まる。

## 調整パラメータ(ソース冒頭)

| 定数 | 値 | 効果 |
|---|---|---|
| `BLOCK` | 17 | ブロック一辺px(TNT・土の共通グリッド) |
| `SPAWN_INTERVAL` | 60ms | バースト時の投下ペース。短いほどドバッと |
| `FUSE_MS` | 900 | タップ着火の導火線 |
| `CHAIN_FUSE` | 120–420ms | 誘爆の導火線。短いほどチェーンが速く爽快 |
| `AUTO_FUSE` | 3.5–7s | 自然発火までの寿命。長いほど「遊べる」時間が増える |
| `BLAST_R` / `CHAIN_R` / `DIRT_DESTROY_R` | 85 / 60 / 64 | 爆風・誘爆・土破壊の半径 |
| `BLAST_FORCE` | 0.045 | 吹き飛ばしの強さ |

## パフォーマンスの注意

- 土は静的ボディ約300個+TNT動的最大160個。Matter.jsでは静的同士の衝突計算は
  発生しないため、負荷の主因は動的ボディ数。重い端末では `block-gain` を上げて
  TNT総数を減らすのが第一手。
- `prefers-reduced-motion` 対応(揺れ・フラッシュの抑制)は今後の課題。
