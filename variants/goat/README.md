# `<goat-progress>` — ヤギが草むらを食べて進む進捗表現(毒草よけゲーム付き)

一面の草むらを、ヤギが**端から順に食べて**進んでいく。食べ進んだ境界が進捗。
草むらにはたまに**紫の毒草**が生え、ヤギがそこまで食べ進むと口にして
**少しずつ具合が悪くなる**(体調メーターが下がり、見た目も緑がかってフラフラに)。
プレイヤーは毒草を**クリック/タップで抜いて**ヤギを守る。

依存ゼロ(canvas のみ)。

## 進捗モデル(このコンポーネントの核)

| 変数 | 意味 | 役割 |
|---|---|---|
| `value` | 実進捗 | 外部から流し込む唯一の入力 |
| `displayValue` | 食べ進み | `value` へ滑らかに追従(読み取り専用) |
| `health` | 体調 0–1 | 毒草を食べると下がる(読み取り専用・演出のみ) |

- ヤギの食べ進み(=表示)は実進捗へ**遅れて**追従するが**嘘はつかない**。
- **体調・毒草は純粋に演出レイヤー**で、進捗(value/displayValue)には一切影響しない。
  具合が悪くてもヤギは止まらず、表示は常に実進捗に正直。

## 毒草よけゲーム

- 毒草はヤギの少し前方にランダムな間隔で生える(`poison-rate` で出やすさ調整、`0`で無し)。
- **クリック/タップで抜く**と取り除ける(`poison-removed` イベント)。
- 抜きそびれてヤギの食べ進み境界が毒草に届くと食べてしまい、体調が
  `SICK_PER_BITE` ぶん悪化(`poison-eaten` イベント)。体調は時間で少しずつ自然回復。
- 体調に応じてヤギの体色が緑がかり、揺れが大きくなり、目が半目→ぐるぐるに変化する。

## 使い方

```html
<script type="module" src="goat-progress.js"></script>

<goat-progress id="gp"></goat-progress>

<script>
  const gp = document.getElementById("gp");
  onServerProgress((pct) => { gp.value = pct; });

  gp.addEventListener("complete", () => { /* 完食 */ });
  gp.addEventListener("poison-eaten", () => console.log("具合が悪く…", gp.health));
  gp.addEventListener("poison-removed", () => console.log("毒草を抜いた!"));
</script>
```

### 属性

| 属性 | 既定値 | 説明 |
|---|---|---|
| `value` | 0 | 実進捗 0–100 |
| `height` | 360 | 高さpx(CSSの height でも可) |
| `demo` | なし | `smooth` / `step`(10%刻み)で自走。**本番では使わない** |
| `duration` | 18000 | demoの所要ms |
| `poison-rate` | 1 | 毒草の出やすさ係数。`0` で毒草なし |

### プロパティ / メソッド / イベント

- `.value` / `.displayValue`(=食べ進み) / `.health`(体調 0–1)
- `.steps` / `.note` — テキスト表示(全バリエーション共通)
- `.reset()` — 進捗・体調・毒草を初期化
- イベント `complete` — 食べ進みが100%に到達(1回)
- イベント `poison-eaten` — ヤギが毒草を口にした
- イベント `poison-removed` — 毒草を抜いて取り除いた

## 実装ノート

- **正直設計**: 食べ進み `displayValue` は `value` へイージング追従するだけ。中央上の
  % は `value` をそのまま表示。体調・毒草はゲーム用の別レイヤー。
- **草むら**: 幅に応じて生成した草ブレードを `t`(0..1)で保持。食べ進み境界
  `frontier = displayValue/100` より左は短い刈り株、右は伸びた草として描く。
- **毒草**: 位置は `t` で保持(リサイズしても比率で追従)。クリック判定は画面座標で
  距離判定。抜くと上へ引き抜かれて消える。
- **ヤギ**: 横向きの自前ベクター。体色を `#mix` で健康色↔病気色に補間し、
  揺れ・目の表情(通常/半目/ぐるぐる)を体調で切り替える。
- **依存ゼロ**: 物理エンジンは使わず、食べ進み・発芽・体調はすべてイージングで表現。
